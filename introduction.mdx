Composio gives your AI agents the ability to act ÔÇö not just reason. With a single SDK call you can connect any agent to **1,000+ real-world integrations** (GitHub, Slack, Gmail, Notion, and many more), complete with managed authentication, ephemeral sessions, event triggers, and full Model Context Protocol (MCP) support.

Both a **TypeScript** package (`@composio/core`) and a **Python** package (`composio`) ship from the same repository and stay in sync, so you can use whichever language your agent stack requires.

## Key Concepts

| Concept | What it is |
|---|---|
| **Tools** | Individual callable actions or read operations exposed by an integration (e.g. `GITHUB_CREATE_ISSUE`). |
| **Toolkits** | Named bundles of related tools for one service (e.g. `github`, `slack`, `gmail`). |
| **Sessions** | Ephemeral, per-user runtime contexts created with `composio.create(userId, ÔÇŽ)` that scope tool access, auth, and MCP endpoints to a single agent run. |
| **Providers** | Thin adapter packages (`@composio/openai-agents`, `composio-langchain`, ÔÇŽ) that wrap Composio tools into the format expected by each AI framework. |
| **Triggers** | Webhook and polling listeners that fire your callback when an external event occurs (e.g. a new GitHub PR, an incoming email). |
| **MCP** | Every session exposes a live [Model Context Protocol](https://modelcontextprotocol.io) endpoint at `session.mcp.url`, letting MCP-compatible clients consume Composio tools natively. |
