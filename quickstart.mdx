title: 'Quickstart: Run Your First Composio Agent Tool Call'
description: 'Learn how to install Composio, get an API key, create a session, connect a toolkit, and run your first AI agent tool call in under 5 minutes.'
---

This guide walks you through the fastest path from zero to a working AI agent that can perform real actions — creating a GitHub issue — using Composio.

<Steps>
  <Step title="Install packages">
    Install the core SDK and the provider adapter for your AI framework. The examples below use **OpenAI Agents**; swap the provider package for whichever framework you prefer.

    <CodeGroup>
      ```bash npm
      npm install @composio/core @composio/openai-agents @openai/agents
      ```

      ```bash yarn
      yarn add @composio/core @composio/openai-agents @openai/agents
      ```

      ```bash pnpm
      pnpm add @composio/core @composio/openai-agents @openai/agents
      ```

      ```bash pip
      pip install composio composio-openai-agents openai-agents
      ```
    </CodeGroup>
  </Step>

  <Step title="Get your API key">
    Sign up or log in at [composio.dev](https://composio.dev) and copy your API key from the dashboard.

    Set it as an environment variable so the SDK picks it up automatically:

    <CodeGroup>
      ```bash macOS / Linux
      export COMPOSIO_API_KEY="your-api-key-here"
      ```

      ```powershell Windows PowerShell
      $env:COMPOSIO_API_KEY = "your-api-key-here"
      ```
    </CodeGroup>

    <Note>
      You can also pass the key directly to the constructor: `new Composio({ apiKey: '...' })` (TypeScript) or `Composio(api_key='...')` (Python). Using an environment variable is recommended so secrets stay out of source code.
    </Note>
  </Step>

  <Step title="Initialize Composio">
    Create a `Composio` instance and attach a provider. The provider controls how tool schemas are formatted for your chosen AI framework.

    <CodeGroup>
      ```typescript TypeScript
      import { Composio } from '@composio/core';
      import { OpenAIAgentsProvider } from '@composio/openai-agents';

      const composio = new Composio({
        provider: new OpenAIAgentsProvider(),
        // apiKey: 'your-api-key', // or set COMPOSIO_API_KEY env var
      });
      ```

      ```python Python
      from composio import Composio
      from composio_openai_agents import OpenAIAgentsProvider

      composio = Composio(
          provider=OpenAIAgentsProvider(),
          # api_key="your-api-key",  # or set COMPOSIO_API_KEY env var
      )
      ```
    </CodeGroup>
  </Step>

  <Step title="Create a session and fetch tools">
    A **session** is an ephemeral, per-user context that scopes tool access and authentication. Pass a stable `userId` (any string that identifies your end-user) and the list of toolkits you want to enable.

    <CodeGroup>
      ```typescript TypeScript
      // Create a session scoped to this user and the GitHub toolkit
      const session = await composio.create('user@acme.org', {
        toolkits: ['github'],
      });

      // Fetch tools wrapped in the format OpenAI Agents expects
      const tools = await session.tools();
      ```

      ```python Python
      # Create a session scoped to this user and the GitHub toolkit
      session = composio.create(
          user_id='user@acme.org',
          toolkits=['github'],
      )

      # Fetch tools wrapped in the format OpenAI Agents expects
      tools = session.tools()
      ```
    </CodeGroup>

    <Tip>
      Tool slugs follow the `TOOLKIT_ACTION` pattern — all uppercase, toolkit name first, then the action name. For example, `GITHUB_CREATE_ISSUE`, `SLACK_SEND_MESSAGE`, or `GMAIL_SEARCH_EMAILS`. You can filter by individual slugs using the `tools` key inside the session config.
    </Tip>
  </Step>

  <Step title="Run your agent">
    Pass the retrieved tools to your agent and run it. Composio handles routing each tool call to the correct API, injecting the user's credentials automatically.

    <CodeGroup>
      ```typescript TypeScript
      import { Composio } from '@composio/core';
      import { OpenAIAgentsProvider } from '@composio/openai-agents';
      import { Agent, run } from '@openai/agents';

      const composio = new Composio({
        provider: new OpenAIAgentsProvider(),
      });

      const session = await composio.create('user@acme.org', {
        toolkits: ['github'],
      });

      const tools = await session.tools();

      const agent = new Agent({
        name: 'GitHub Agent',
        instructions: 'You are a helpful GitHub assistant.',
        tools,
      });

      const result = await run(
        agent,
        'Create a GitHub issue titled "Test from Composio" in my repo.',
      );

      console.log(result.finalOutput);
      ```

      ```python Python
      import asyncio
      from composio import Composio
      from composio_openai_agents import OpenAIAgentsProvider
      from agents import Agent, Runner

      composio = Composio(provider=OpenAIAgentsProvider())

      session = composio.create(
          user_id='user@acme.org',
          toolkits=['github'],
      )

      tools = session.tools()

      agent = Agent(
          name='GitHub Agent',
          instructions='You are a helpful GitHub assistant.',
          tools=tools,
      )

      async def main():
          result = await Runner.run(
              starting_agent=agent,
              input='Create a GitHub issue titled "Test from Composio" in my repo.',
          )
          print(result.final_output)

      asyncio.run(main())
      ```
    </CodeGroup>
  </Step>
</Steps>
