This page is the complete reference for every `composio` CLI command. All commands accept `--help` for inline usage information and `--json` for machine-readable output.

<Note>
  Add `--json` to any command to receive structured JSON output instead of the interactive table view. This is recommended for CI pipelines and shell scripting.
</Note>

## Authentication

### `composio login`

Open a browser to authenticate and store credentials locally.

```bash
composio login
composio login --no-browser          # Print URL, don't open browser
composio login --user-api-key <key>  # Non-interactive login with API key
composio login --org <id|name>       # Set active org during login
```

### `composio logout`

Remove stored credentials from the local machine.

```bash
composio logout
```

### `composio signup`

Create a new Composio account from the CLI.

```bash
composio signup
```

### `composio whoami`

Display the currently authenticated account (email, organisation, project, account type).

```bash
composio whoami
```

## Tools

### `composio tools list <toolkit>`

List tools available in a toolkit.

```bash
composio tools list github
composio tools list gmail --query "send email"
composio tools list slack --limit 50
```

| Argument / Flag | Description |
|---|---|
| `<toolkit>` | Toolkit slug (e.g. `github`, `gmail`). **Required.** |
| `--query <text>` | Text search by name, slug, or description. |
| `--tags <tag>` | Filter by tag (e.g. `important`). |
| `--limit <n>` | Results per page (1–1000, default 30). |

### `composio tools search <query>`

Semantic search for tools across all toolkits.

```bash
composio tools search "send a slack message"
composio tools search "create github issue" --limit 10
```

### `composio tools info <slug>`

Show detailed metadata for a single tool: description, input schema, output schema, and required scopes.

```bash
composio tools info GITHUB_STAR_A_REPOSITORY_FOR_AUTHENTICATED_USER
```

### `composio tools execute <slug>`

Execute a tool interactively from the CLI. The CLI validates input arguments, resolves the active connected account, and prints the result.

```bash
composio tools execute GITHUB_STAR_A_REPOSITORY_FOR_AUTHENTICATED_USER
composio tools execute GMAIL_SEND_EMAIL --json
```

## Connected Accounts

### `composio connected-accounts list`

List all connected accounts in your project.

```bash
composio connected-accounts list
composio connected-accounts list --json
```

### `composio connected-accounts info <id>`

Show details for a specific connected account.

```bash
composio connected-accounts info ca_abc123
```

### `composio connected-accounts link <toolkit>`

Initiate an OAuth connection flow for a toolkit. Opens the auth URL in your browser (or prints it with `--no-browser`).

```bash
composio connected-accounts link github
composio connected-accounts link gmail --no-browser
```

### `composio connected-accounts whoami`

Show which connected accounts are active for the current CLI session user.

```bash
composio connected-accounts whoami
```

## Triggers

### `composio triggers list`

List all trigger types available, optionally filtered by toolkit.

```bash
composio triggers list
composio triggers list --toolkit github
```

### `composio triggers create <slug>`

Create (or upsert) a trigger instance interactively.

```bash
composio triggers create GITHUB_COMMIT_EVENT
```

### `composio triggers enable <id>`

Enable a trigger instance by ID.

```bash
composio triggers enable ti_abc123
```

### `composio triggers disable <id>`

Disable a trigger instance by ID.

```bash
composio triggers disable ti_abc123
```

### `composio triggers listen`

Start a real-time listener in the terminal that prints trigger events as they arrive.

```bash
composio triggers listen
composio triggers listen --trigger-slug GITHUB_COMMIT_EVENT
composio triggers listen --user-id user_alice
```

### `composio triggers info <slug>`

Show the schema and configuration options for a trigger type.

```bash
composio triggers info GITHUB_COMMIT_EVENT
```

### `composio triggers status`

Show a summary of all active trigger instances in your project.

```bash
composio triggers status
```

## Code Generation

### `composio generate ts`

Generate TypeScript type stubs for toolkits, tools, and triggers.

```bash
composio generate ts --toolkits github --output-dir ./src/composio-types/
composio generate ts --toolkits github --toolkits gmail --output-dir ./types/
```

| Flag | Description |
|---|---|
| `--toolkits <slug>` | Toolkit to generate types for. Repeatable (`--toolkits gmail --toolkits slack`). |
| `--output-dir <path>` | Output directory path (alias: `-o`). |
| `--compact` | Emit a single TypeScript file instead of one file per toolkit. |
| `--transpiled` | Also emit transpiled JavaScript alongside TypeScript files. |

### `composio generate py`

Generate Python type stubs for toolkits.

```bash
composio generate py --toolkits github --output-dir ./composio_types/
```

| Flag | Description |
|---|---|
| `--toolkits <slug>` | Toolkit to generate types for. Repeatable (`--toolkits gmail --toolkits slack`). |
| `--output-dir <path>` | Output directory path (alias: `-o`). |

## Development Utilities

### `composio dev`

Start the Composio local development server. Enables hot-reloading of local tool definitions and proxied API calls for rapid iteration.

```bash
composio dev
```

### `composio proxy`

Proxy an authenticated API request through a connected account directly from the CLI. Provide a toolkit slug (`--toolkit`) to select which connected account's credentials are injected into the request.

```bash
composio proxy https://api.github.com/user --toolkit github
composio proxy https://api.github.com/user --toolkit github --method GET
composio proxy https://api.example.com/data --toolkit myapp -X POST -d '{"key":"value"}'
```

| Argument / Flag | Description |
|---|---|
| `<url>` | Absolute or relative API endpoint to call. **Required.** |
| `--toolkit <slug>`, `-t` | Toolkit slug whose connected account credentials are used. **Required.** |
| `--method <method>`, `-X` | HTTP method: `GET`, `POST`, `PUT`, `DELETE`, `PATCH`. Defaults to `GET`. |
| `--header <Name: value>`, `-H` | Request header in `"Name: value"` format. Repeatable. |
| `--data <body>`, `-d` | Request body as raw text, JSON, `@file`, or `-` for stdin. |
